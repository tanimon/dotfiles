# Declarative Marketplace Sync

## Overview

Replace the 3-layer marketplace sync system (`modify_` + `.data` + `run_after_` reverse sync) with a declarative plain text list and a `run_onchange_` script. Also remove `installed_plugins.json` from chezmoi management entirely.

## Problem Statement

The current system uses bidirectional sync to manage `known_marketplaces.json` across machines:
1. `.data` files — shadow copies with `{{ .chezmoi.homeDir }}` template substitution
2. `modify_` scripts — seed on new machines, merge on existing
3. `run_after_20-sync-plugins.sh.tmpl` — reverse-syncs live files back to `.data`
4. `run_after_10-ensure-marketplaces.sh.tmpl` — runs `claude plugin marketplace add` for missing directories

This is complex, has template escaping issues, and produces JSON merge conflicts across machines. The user only needs marketplace *registrations* synced — not plugin install/enable state.

## Proposed Solution

A unidirectional, declarative approach (see brainstorm: `docs/brainstorms/2026-03-08-marketplace-sync-brainstorm.md`):

- **`dot_claude/plugins/marketplaces.txt`** — one marketplace source per line (`owner/repo` or full URL)
- **`scripts/update-marketplaces.sh`** — generates `marketplaces.txt` from `claude plugin marketplace list --json`
- **`run_onchange_after_add-marketplaces.sh.tmpl`** — triggered when `marketplaces.txt` changes; runs `claude plugin marketplace add` for each entry
- **Remove** all `modify_`, `.data`, and reverse-sync files for both `known_marketplaces.json` and `installed_plugins.json`
- **Add** `installed_plugins.json` and `known_marketplaces.json` to `.chezmoiignore`

Key insight: `*.txt` is already in `.chezmoiignore`, so `marketplaces.txt` won't deploy to `~/` but is readable via `{{ include }}` for hash tracking. `claude plugin marketplace add` is idempotent — adding an already-registered marketplace is a no-op.

## MVP

### Phase 1: Create new declarative files

#### `scripts/update-marketplaces.sh`

Generates `marketplaces.txt` from current `claude plugin marketplace list --json` output. Workflow: add a marketplace locally via `claude plugin marketplace add`, then run this script to update the list, commit, and push.

```bash
#!/usr/bin/env bash
set -euo pipefail

SCRIPT_DIR="$(cd "$(dirname "$0")" && pwd)"
REPO_ROOT="$(cd "$SCRIPT_DIR/.." && pwd)"
TARGET="${REPO_ROOT}/dot_claude/plugins/marketplaces.txt"

if ! command -v claude &>/dev/null; then
  echo "error: claude CLI not found" >&2
  exit 1
fi

if ! command -v jq &>/dev/null; then
  echo "error: jq not found" >&2
  exit 1
fi

claude plugin marketplace list --json \
  | jq -r '.[] | if .source == "github" then .repo else .url end' \
  | sort \
  > "$TARGET"

echo "Updated ${TARGET} ($(wc -l < "$TARGET" | tr -d ' ') marketplaces)"
```

#### `dot_claude/plugins/marketplaces.txt`

Initial content (generated by the script above):

```text
EveryInc/compound-engineering-plugin
OthmanAdi/planning-with-files
affaan-m/everything-claude-code
anthropics/claude-plugins-official
anthropics/skills
jarrodwatts/claude-delegator
```

#### `.chezmoiscripts/run_onchange_after_add-marketplaces.sh.tmpl`

```bash
{{ if eq .chezmoi.os "darwin" -}}
#!/usr/bin/env bash
set -euo pipefail

# marketplaces.txt hash: {{ include "dot_claude/plugins/marketplaces.txt" | sha256sum }}

if ! command -v claude &>/dev/null; then
  echo "claude CLI not found, skipping marketplace registration"
  exit 0
fi

while IFS= read -r source || [ -n "$source" ]; do
  # Skip empty lines and comments
  [[ -z "$source" || "$source" == \#* ]] && continue
  echo "Ensuring marketplace: ${source}"
  claude plugin marketplace add "$source" || true
done < "{{ .chezmoi.sourceDir }}/dot_claude/plugins/marketplaces.txt"
{{ end -}}
```
